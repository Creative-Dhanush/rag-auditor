RAG pipeline architecture
End-to-end flow from document ingestion to grounded answer generation via free Google Gemini API.
Ingestion pipeline
Offline — runs locally
📄 PDF extraction
PDF.js extracts raw text page by page, preserving structure. Metadata: filename, page, char offset.
→
✂️ Chunking
512-token sliding window, 50-token overlap. Each chunk tagged with doc_id, page, chunk_index.
→
🧮 TF-IDF Embedding
In-browser TF-IDF vectorization + cosine similarity for search. No API cost, runs entirely client-side.
→
🗄️ Vector store
In-browser JS vector store with metadata. Supports upsert, delete, and targeted page-level operations.
Query pipeline
Live — runs on every question
🔒 Input guardrail
Regex + pattern classifier blocks prompt injections, jailbreaks, and out-of-scope queries before retrieval.
→
🔍 Vector search
TF-IDF cosine similarity retrieves top-50 candidate chunks from the in-memory store.
→
↕️ Reranking
BM25-style keyword overlap reranker filters 50→8 chunks. Only highest-precision context reaches the LLM.
🧠 Gemini generation
Structured prompt enforces grounded answering from context only. Gemini 2.0 Flash generates the final answer.
→
✅ Output guardrail
Checks every claim against retrieved chunks. Unverified claims flagged. On API failure: raw chunks shown.
Token optimization
• Top-8 reranking saves ~84% tokens vs sending all 50 chunks
• Chunk stripping removes headers/footers
• max_tokens=1024 hard cap on generation
• Context window optimized — only relevant chunks sent
Chunking strategy
• 512-token window, 50-token overlap
• Page-level metadata preserved
• Sentence-boundary aware splitting
• Targeted upsert/delete by chunk ID
Failover strategy
• try/catch wraps every API call
• Timeout: 30s then fallback fires
• Raw reranked chunks shown instead
• Warning banner shown to user

Fallback code pattern
async function ragQuery(question, chunks) {
  try {
    const answer = await callGeminiAPI(question, chunks);
    return { type: 'ai', answer, chunks };
  } catch (err) {
    console.warn('API failed:', err.message);
    return {
      type: 'fallback',
      warning: `AI offline (${err.message})`,
      chunks
    };
  }
}
